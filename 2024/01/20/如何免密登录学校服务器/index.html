1. 第一步安装PuTTY

首先我们需要在windows上面下载 PuTTY. 苹果用户可以使用 sshpass. 下载地址在这 Download PuTTY: latest release (0.80) (greenend.org.uk)

然后找到属于自己的安装包, 比如我的电脑是 64位系统, 所以选择 64-bit x86 的安装包. 双击下载后的安装包, 跟着步骤开始安装.

找到下载后的安装包, 双击打开. 记得选择下载位置. 我一般不放在C盘. 毕竟C盘且用且珍惜.

然后一直按next, 等待安装结束就好啦…

如何检测安装是否成功, 可以打开cmd然后敲plink命令, 看看是否能正确显示

原理解释:

书接上回, 我们使用用户名+密码 的方式登录门户服务器, 然后使用 密匙 来免密登录学校服务器, 实现一次密码直接登陆的”壮举”. 但是, 我们有没有能力直接将密码携带在命令里登陆呢, 那样是不是就不用输入密码了? 有搞头, 冲!.

在网上搜了好一番. 才发现ssh并没有内置的, 携带密码的选择. 所以我们得使用一些外挂, 也就是PuTTY. 一个小巧简洁的ssh登录客户端. 它里面提供了一个工具 plink 可以在命令行里携带密码发送 ssh请求. 这不就是我们需要的东西嘛

现在携带密码的工具已经搞定了, 那我们应该如何在使用ssh命令时, 让其自动执行呢? 记不记得上回在配置文件里设置的 ProxyJump 参数, 它接收一个 username@address 字符串, 然后在连接到目的服务器前, 先使用ssh连接到 ProxyJump 参数里指定的服务器, 然后再通过那个服务器作为代理发送ssh请求到目标服务器. 但是问题来了: ProxyJump只能发送ssh请求, 没法使用plink来携带密码怎么办? . 于是一个新的参数出现了, ProxyCommand 它允许指定用于连接到服务器的命令. 也就是说, 我们可以使用 plink 来连接我们的门户服务器了.

下面是对配置完整的解释:

  • 使用 plink -ssh -l h3626454 -pw <password> gatekeeper.cs.hku.hk 来连接代理服务器,
  • -ssh: 指定使用ssh登录
  • -l : 指定登录用户
  • -pw: 指定携带的密码
  • -nc %h:%p 命令使用 nc (netcat) 工具设置转发请求的目的服务器地址以及端口
  • %h 和 %p 分别是 host 和 port 的占位符, 在命令执行时会自动替换成配置前面设置的 HostName 和 Port 的值

Remark: ProxyCommand 只能用于连接代理服务器, 不能直接连接目标服务器. 所以我们一般搭配 -nc 参数来配置最终的目标服务器.

remark:

  • 在电脑上必须安装 PuTTY, 要求在终端内输入 plink 不报错, 即命令存在. 如命令不存在, 先检查有无下载 PuTTY, 再检查环境变量是否配置. 如果都没出错, 可以考虑删了重下
  • 如果原先已经在 ~/.ssh/config 下配置过类似配置, 或是运行过一次脚本, 需要自己手动清除之前的配置. 因为新配置默认添加在文件末尾. 如果前面有匹配的配置, 就会覆盖新的配置!
  • 由于第二次登录学校服务器是使用ssh密匙实现的! 所以仍旧需要跟着 “如何快速登录学校服务器” 里的第二步在学校服务器的用户目录下创建 .ssh/authorized_keys 并配置公匙
  • 如果以上三点都没问题, 但是还是没法一键登录学校服务器, 可以检查用户密码是否填写错误. 最后实在搞不定可以私聊我.

使用教程如图:

  1. 双击 .bat 文件开始运行脚本
  2. 输入用户名, 密码, 以及自定义的主机名
  3. 脚本会自动在 ~/.ssh/config 中生成配置

有些人觉得这么大费周章其实很没必要. 但其实这个具有很广泛的用途. 在这几天专研的过程中, 我发现这种 跳板机-内网服务器 的形式在很多公司里都有应用. 公司不会轻易的将自己的服务器暴露在外网, 而是使用一个 跳板机 来充当内网访问的门户, 这样做的好处有以下几点:

  • 安全性增强： 跳板机作为中介可以减少直接暴露内部服务器给外部的风险，提高网络安全性。比如防御黑客入侵, DDos等
  • 审计和监控： 通过跳板机，管理员可以更容易地监控用户的活动和进行审计，因为所有的连接都经过这个服务器。
  • 访问控制： 跳板机可以实施更严格的访问控制，只允许授权用户连接到内部服务器。
  • 集中管理： 管理员可以集中管理跳板机上的安全策略，而无需在每个服务器上进行配置。

所以通过这种可以越过跳板机直接访问内网服务器的方式, 一定程度上的减少了浪费的时间.

注意:

由于学校的跳板机没有给我们权限更改文件, 所以只能使用 密码 + ssh密匙 混合的形式来登录. 但是以后如果在公司, 更推荐的做法是找你们的管理员把你的密匙也添加到 跳板机 上, 这样就可以直接使用 密匙登录 , 相比起将密码写死在文件内的方式更加的安全.
