快速登录学校服务器第二版: 一步到”胃”

俗话说的好, 懒惰是进步的第一生产力. 咱们上一讲聊到如何自定义服务器地址从而避免记忆冗长的地址, 如何做到只输入一次密码就可以直接登录. 但是! 密码太长太恶心了啊QAQ. 于是就有了这篇, 偷懒大法. 咱直接一步到位, 配置一次, 以后无需再填写任何密码. 一条命令直接登录! 废话不多说, 开整!!!

1. 第一步安装PuTTY

首先我们需要在windows上面下载 PuTTY. 苹果用户可以使用 sshpass. 下载地址在这 Download PuTTY: latest release (0.80) (greenend.org.uk)

然后找到属于自己的安装包, 比如我的电脑是 64位系统, 所以选择64-bit x86 的安装包

找到以后下载安装包, 然后双击打开. 记得选择下载位置. 我一般不放在C盘. 毕竟C盘且用且珍惜

然后一直按next, 等待安装结束就好啦…

如何检测安装是否成功, 可以打开cmd然后敲plink命令, 看看是否能正确显示

如果报错的话可以自己找一下如何配置windows环境变量. 把plink.exe所处的文件夹路径添加到PATH里就可以啦

好了, 接下来是解释时间. 这玩意搞了我挺久的说实话… 要个三连 (啊呸). 什么三连. 做好事不留名好吧.

原理解释:

书接上回, 我们使用用户名+密码 的方式登录门户服务器, 然后使用 密匙 来免密登录学校服务器, 实现一次密码直接登陆的”壮举”. 但是, 我们有没有能力直接将密码携带在命令里登陆呢, 那样是不是就不用输入密码了? 所以, 开搞.

在网上搜了好一番. 才发现ssh并没有内置的, 携带密码的选择. 所以我们得使用一些外挂, 也就是PuTTY. 一个小巧简洁的ssh登录客户端. 它里面提供了一个工具 plint 可以在命令行里携带密码发送 ssh请求. 于是乎我找上门了.

现在携带密码的工具已经搞定了, 如何在使用ssh命令时, 让其自动执行呢? 记不记得上回在配置文件里设置的 ProxyJump 参数, 它接收一个 username@address 字符串, 然后在连接到目的服务器前, 先使用ssh连接到ProxyJump参数里指定的服务器, 然后再通过那个服务器作为代理发送ssh请求到目标服务器. 但是问题来了: ProxyJump只能发送ssh请求, 没法使用plint来携带密码怎么办? . 于是一个新的参数出现了, ProxyCommand 它允许指定用于连接到服务器的命令. 也就是说, 我们可以使用非ssh来连接我们的门户服务器了.

这就是对以下配置完整的解释:

  • 使用 plink -ssh -l h3626454 -pw <password> gatekeeper.cs.hku.hk 来连接代理服务器,
  • -nc %h:%p 命令使用 nc (netcat) 工具设置转发请求的目的服务器地址以及端口
  • %h 和 %p 分别是 host 和 port 的占位符, 在命令执行时会自动替换成配置前面设置的 HostName 和 Port 的值

按照官方ssh配置里对于 ProxyCommand 的解释, 我们应该是能使用任何非ssh命令来连接远程服务器的, 但是不知道为什么我期待的使用plint来发起远程连接的配置并没有生效. 如果有了解的大佬可以跟我讲讲嘛
